# 空白毕业典礼 (Blank Graduation)
## 基于文件系统的互动小说 - 开发文档

---

## 项目概述

**类型**: 交互式叙事游戏 (Interactive Fiction)
**核心机制**: 文件系统 + Git 分支作为时空逻辑
**目标**: 玩家通过探索文件夹层级、阅读 Markdown 文件、运行 Python 脚本做选择，体验一个"失忆毕业典礼"的百合故事

---

## 世界观与故事设定

### 核心设定
- **玩家角色**: 即将毕业的大学生，醒来时丢失了大学四年的所有记忆
- **另一位女主 (小夏)**: 从大一就陪伴在玩家身边的存在
- **时间线**:
  - 现在（毕业典礼当天）：玩家处于失忆状态
  - 过去（记忆碎片）：通过探索逐步还原

### 故事走向
- **悬念**: 为什么玩家会失忆？小夏为什么小心翼翼？
- **记忆线**: 通过探索文件找回甜蜜记忆
- **真相线**: 触及核心事件——记忆被封印的真正原因

### 两个结局
1. **BE (致郁)**: 四年是自我构建的"温柔乡"，现实太过残酷。记忆回来了，但"她"不在了
2. **Sweet GE (甜蜜)**: 玩家经历了创伤，但小夏一直守护。现在可以重新开始

### Meta 感
只在最后时刻：玩家发现故事有作者署名，是自己的名字

## 技术架构

### 目录结构
```
game/                          # 最终可运行的游戏文件夹
├── Scene.md                  # 当前场景描述
├── Story.md                  # 当前故事情节（只读）
├── Note.md                   # 玩家收集的线索
├── Choice.py                 # 交互式选择脚本（引擎）
├── nodes/                    # 所有节点文件
│   ├── node_01/
│   ├── node_02/
│   └── ...
└── .git/                     # Git 时空引擎

dev_script/                    # 开发阶段的源文件
├── nodes/                    # 各节点原始文件
│   ├── node_01/
│   ├── node_02/
│   └── ...
└── branches/                 # 分支结构定义（预留）
```

### Git 时空逻辑
- **同一分支的 commit**: 代表时间推进
- **不同 branch**: 代表平行时空/选择结果
- **切换分支**: `git checkout branch_name`
- **时间回溯**: `git checkout commit_hash`
- **选择推进**: 玩家通过 Choice.py 选择后，脚本复制对应节点文件到根目录

### 核心文件说明
| 文件 | 作用 |
|------|------|
| `Scene.md` | 描述玩家当前所处的"位置"和所见所闻 |
| `Story.md` | 叙述当前发生的事件（只读） |
| `Note.md` | 玩家收集的线索、提示、物品描述 |
| `Choice.py` | 双击运行进行选择，或用 `python Choice.py --choice N` |

## 交互方式

### 基本规则
1. 玩家双击 `Choice.py` 或运行 `python Choice.py`
2. 脚本显示当前可选选项
3. 玩家输入数字选择
4. 脚本复制对应节点文件到根目录

### 命令行用法
```bash
python Choice.py                      # 交互式选择
python Choice.py --choice <N>         # 非交互式选择
python Choice.py --status             # 查看当前状态
python Choice.py --goto <node_id>     # 跳转到指定节点（测试用）
```

### 玩家禁止行为（防止作弊）
- 禁止使用 `git checkout` 自行切换分支
- 禁止使用 `grep` 搜索整个仓库
- 禁止修改 `.md` 文件（Story.md 应设为只读）
